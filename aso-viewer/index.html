Bugs identifiés — 7 problèmes bloquants

BUG #1 Couleurs obsolètes dans themes.xml app/src/main/res/values/themes.xml

Le fichier themes.xml actuel pointe vers @color/primary, @color/primary_dark, @color/accent définis comme #1F2937, #111827, #3B82F6. Aucun style AsoText.* ni AsoButton.* n'est présent. Toute l'UI sera grise/bleue au lieu du thème noir/vert.

- <color name="primary">#FF1F2937</color> - <color name="accent">#FF3B82F6</color> + <color name="aso_bg_base">#0E0F11</color> + <color name="aso_accent">#4ADE80</color>
BUG #2 themes.xml sans styles AsoText.* et AsoButton.* app/src/main/res/values/themes.xml

Les layouts du design ZIP référencent style="@style/AsoText.Mono", style="@style/AsoText.MonoLabel", style="@style/AsoButton.Secondary", style="@style/AsoButton.Critical". Ces styles n'existent pas dans le code actuel → build error ou crash runtime.

// À ajouter dans themes.xml : + <style name="AsoText.Mono" parent=""> + <item name="android:fontFamily">monospace</item> + <item name="android:textSize">@dimen/aso_text_sm</item> + <item name="android:letterSpacing">0.05</item> + </style>
BUG #3 dimens.xml absent — aucun token aso_space_* ni aso_text_* app/src/main/res/values/dimens.xml

Le projet n'a pas de fichier dimens.xml. Tous les layouts du design ZIP utilisent @dimen/aso_space_4, @dimen/aso_radius_lg, @dimen/aso_text_display, etc. — ce qui provoquerait une erreur de compilation immédiate.

BUG #4 Drawables aso_card_bg, aso_led_*, aso_capture_bg absents app/src/main/res/drawable/

Les 5 drawables du design ZIP sont absents du dossier drawable/ du projet. Les layouts du ZIP y font référence via android:background="@drawable/aso_card_bg" etc. → aapt error à la compilation.

BUG #5 strings.xml incomplet — 15+ clés manquantes app/src/main/res/values/strings.xml

Le strings.xml actuel ne contient pas les nouvelles clés du design :

manquant : app_brand_a, app_brand_so, app_tagline manquant : home_status_gnss, home_status_calib, home_status_storage manquant : home_status_to_check, start_scan_sub, calibrate_sub manquant : my_scans, my_scans_sub, capture_rec manquant : ind_feat, hud_btn_note, alert_*, review_*
BUG #6 activity_capture.xml — ConstraintLayout incomplet pour le HUD indicators app/src/main/res/layout/activity_capture.xml

Le layout du design ZIP pour hud_indicators utilise des LinearLayout imbriqués avec android:background="@drawable/aso_card_bg" sur chaque chip. Le code actuel utilise une rangée horizontale plate sans card. Résultat visuel complètement différent.

BUG #7 MetadataWriter — import BarometerSample du mauvais package app/src/main/java/ch/aso/capture/storage/MetadataWriter.kt

MetadataWriter.kt importe ch.aso.capture.logger.BarometerSample mais ce modèle est défini dans ch.aso.capture.logger.SampleModels.kt sous le nom BarometerSample. Le fichier SampleModels.kt ne déclare pas GnssSample au bon package non plus — cross-import incohérent.

- import ch.aso.capture.logger.BarometerSample // inexistant - import ch.aso.capture.logger.GnssSample // inexistant + import ch.aso.capture.logger.SensorSample + import ch.aso.capture.logger.GnssSample // défini dans SampleModels.kt // vérifier que tous les data class sont dans SampleModels.kt

Avertissements — 5 problèmes non-bloquants

WARN #1 Police Sora + JetBrains Mono non intégrées (prévues v0.2) app/src/main/res/font/

Le DESIGN_SYSTEM.md indique que l'intégration des fichiers .ttf Sora et JetBrains Mono est prévue pour v0.2. Actuellement le dossier res/font/ est vide. L'app utilise les polices système (sans-serif-light, monospace). Pas bloquant pour v0.1 mais l'identité visuelle finale sera différente.

WARN #2 aso_status_ok et aso_accent ont la même valeur app/src/main/res/values/colors.xml (design)

Dans le design ZIP, aso_status_ok et aso_accent sont tous les deux #4ADE80. C'est probablement intentionnel (vert = ok = brand) mais cela signifie qu'une barre de progression accent et un LED "ok" sont visuellement identiques, sans distinction possible en état d'alerte partielle.

WARN #3 coverage_bar ProgressBar — progressTint Material3 déprécié app/src/main/res/layout/activity_capture.xml (design)

Le layout design utilise android:progressTint et android:progressBackgroundTint sur un ProgressBar standard. Avec Material3, cela est souvent ignoré ou produit un rendu inattendu. À remplacer par un LinearProgressIndicator Material3 pour garantir la cohérence.

WARN #4 action_primary dans activity_main.xml sans ConstraintLayout constraints app/src/main/res/layout/activity_main.xml (design)

Le LinearLayout#action_primary n'a que constraintTop_toBottomOf="@id/status_card" mais pas de constraintStart_toStartOf ni constraintEnd_toEndOf. Dans un ConstraintLayout, sans ces deux contraintes horizontales, la vue ne sera pas positionnée correctement sur tous les appareils.

+ app:layout_constraintStart_toStartOf="parent" + app:layout_constraintEnd_toEndOf="parent"
WARN #5 aso_border_width_default = 1.5dp non supporté sur tous les écrans app/src/main/res/values/dimens.xml (design)

Une valeur de 1.5dp est définie pour aso_border_width_default. Sur les écrans mdpi (1px/dp), cette valeur sera arrondie à 2px. Sur les écrans xxhdpi (3px/dp), elle sera arrondie à 5px — rendu potentiellement épais. À tester sur différentes densités.

Guide d'intégration des fichiers thèmes

Pour intégrer complètement le design system du ZIP dans le projet Android, il faut effectuer ces opérations dans l'ordre. Chaque étape résout un ou plusieurs bugs identifiés.

Etape 1 — Remplacer colors.xml

1
Copier aso-design/app/src/main/res/values/colors.xml vers app/src/main/res/values/colors.xml en écrasant le fichier existant.
2
Vérifier que les anciennes clés primary, primary_dark, accent, red_critical, green_ok, orange_warning sont conservées pour la compatibilité Material3 (elles sont présentes dans le ZIP).

Etape 2 — Créer dimens.xml

1
Copier aso-design/app/src/main/res/values/dimens.xml vers app/src/main/res/values/dimens.xml (fichier absent, à créer).
2
Ce fichier est requis par tous les layouts du design. Sans lui → erreur de compilation immédiate.

Etape 3 — Remplacer themes.xml

1
Copier aso-design/app/src/main/res/values/themes.xml en écrasant l'existant. Ce fichier ajoute les styles AsoText.* et AsoButton.* indispensables.
2
Vérifier que Theme.AsoCapture est toujours référencé dans AndroidManifest.xml — c'est le cas, pas de changement nécessaire.

Etape 4 — Ajouter les drawables

1
Copier les 5 fichiers aso_*.xml depuis aso-design/app/src/main/res/drawable/ vers app/src/main/res/drawable/.
2
Ces drawables référencent les tokens @color/aso_bg_surface etc. — ils nécessitent que l'étape 1 soit faite avant.

Etape 5 — Mettre à jour strings.xml

1
Fusionner le strings.xml du ZIP avec l'existant. Ajouter toutes les clés app_brand_a, home_status_*, capture_rec, hud_*, alert_*, review_*.
2
Ne pas supprimer les clés existantes pour ne pas casser les layouts actuels pendant la migration.

Etape 6 — Remplacer les layouts

1
Copier activity_main.xml et activity_capture.xml du ZIP en écrasant les fichiers existants.
2
Mettre à jour MainActivity.kt et CaptureActivity.kt pour binder les nouveaux IDs : status_card, action_primary, hud_indicators, btn_gcp, btn_burst, btn_stop, btn_scale, btn_note.
3
Corriger le Bug #4 (constraints manquantes) avant de pusher.

Ce qu'il manque encore pour une intégration complète

  • REQUIS Fichier dimens.xml — absent du projet, bloquant
  • REQUIS Polices Sora-Light.ttf, Sora-Black.ttf dans res/font/ — pour le branding A·SO définitif
  • REQUIS Police JetBrainsMono-Regular.ttf — pour tous les styles AsoText.Mono*
  • REQUIS Mise à jour des imports dans MainActivity.kt et CaptureActivity.kt pour les nouveaux IDs de vues
  • REQUIS Correction du package import dans MetadataWriter.kt (Bug #7)
  • OPTIONNEL v0.2 Icône lanceur ic_launcher au format ASO (actuellement icône Android générique)
  • OPTIONNEL v0.2 LinearProgressIndicator Material3 au lieu du ProgressBar standard (Warn #3)
  • OPTIONNEL v0.2 Mode clair adaptatif prévu dans le DESIGN_SYSTEM.md
